<api 에러연구 : http와 https의 차이점>

로컬에서는 http로 보낸 api 요청이 문제없이 먹혔는데
deploy한 곳에서는 api를 받아오지 못하는 상태가 발생
https여야 한다는 에러문구가 뜨길래 http->https로 바꿔줬더니 잘 작동했음
따라서 이참에 아무생각 없이 쓰던 http와 https의 차이에 대해 정리한다
==============================================
<http와 https의 차이점>

http : 하이퍼텍스트 전송 프로토콜의 약자
https : 거기에 secure을 추가하여 보안을 강화한것
         (요즘은 https를 늘리려는 추세지만 아직까진 http도 많음)

http 특징 : 구형, 보안에 취약, 빠르다
https 특징 : 신형, 보안에 강력, 느리다 (체감할 정도는 아님)
==============================================
<https에서 보안성이 강화되는 원리>

데이터를 그대로 보내는 http와 달리
https를 사용할 경우 모든 데이터를 암호화해서 전송하며
전송된 데이터가 바뀌는 것을 보호해준다


https의 암호화에는 SSL이 사용된다
SSL : (Secure Sockets Layer) TCP/IP통신에 사용되는 규약으로
       SSL에서는 2가지 암호화 방식이 있는데 
       https에서는 키교환에는 비대칭키 방식, 실제 데이터 전송에는 대칭키 방식을 사용한다

비대칭키 방식 : 2개의 키를 사용하는 방식으로
                   둘중 하나는 자신이 가지고(개인키)
                   나머지 하나는 상대방에게 공개하여(공개키)
대칭키 방식 : 1개의 키만을 사용한다


키교환 방식은 다음과 같은 순서로 이루어진다
1. 클라이언트가 서버에 접속
2. 서버는 자신의 공개키를 클라이언트에게 전송
3. 클라이언트가 보내온 정보는 암호화 되어있고
   서버는 자신의 개인키로 이를 열람해볼 수 있다
==============================================
<http는 안되고 https는 가능했던 이유 추측>

이번에 사용했던 api요청은
사이트에서 내가 발급받은 인증키를 이용해서 api를 받아오는 방식이기 때문에
인증키의 보안성 문제때문에 http는 안되게 하는것이 아닐까 추측한다
유료 api를 결제했을 경우 구매자의 인증키는 자신만 쓸 수 있도록 지켜줘야하기 때문일것같음
실제 이유는 검색해봐도 잘 안나와서 모르겠다 http를 막아놓을 수 있는건가
